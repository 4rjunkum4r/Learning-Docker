# 17.1 - Basic Docker Commands (Part 1)

In this section, I learned the basic Docker commands used to verify Docker, inspect the Docker Engine, download images, and view images or containers stored on the local machine.

These are the first commands every Docker user should know.

---

# 1. `docker version` / `docker -v`

## What does it do?

Displays the installed Docker version.

- `docker version` shows both the **Docker Client** and **Docker Server (Docker Engine)** versions.
- `docker -v` (or `docker --version`) displays only the installed Docker version.

---

## Syntax

```bash
docker version
```

or

```bash
docker -v
```

---

## Examples

```bash
docker version
```

```bash
docker -v
```

---

## Remember

- Use this command to verify that Docker is installed correctly.
- `docker version` provides more detailed information than `docker -v`.

---

# 2. `docker info`

## What does it do?

Displays detailed information about the Docker Engine running on your system.

It shows information such as:

- Number of containers
- Number of images
- Storage Driver
- Logging Driver
- CPU
- Memory
- Docker Root Directory
- Operating System
- Docker Version

---

## Syntax

```bash
docker info
```

---

## Example

```bash
docker info
```

---

## Remember

- Useful for troubleshooting Docker.
- Gives detailed information about the Docker daemon and Docker host.

---

# 3. `docker pull`

## What does it do?

Downloads a Docker image from a Docker Registry.

By default, Docker downloads images from **Docker Hub**.

If no tag (version) is specified, Docker automatically downloads the **latest** version.

---

## Syntax

```bash
docker pull IMAGE[:TAG]
```

or

```bash
docker pull [OPTIONS] IMAGE[:TAG|@DIGEST]
```

---

## Examples

### Download the latest Ubuntu image

```bash
docker pull ubuntu
```

Docker automatically treats it as:

```text
ubuntu:latest
```

---

### Download a specific version

```bash
docker pull ubuntu:24.04
```

Downloads only Ubuntu version **24.04**.

---

### Download all available tags

```bash
docker pull --all-tags ubuntu
```

Downloads every available version (tag) of the Ubuntu image.

---

## What I Observed

Initially, my system contained only one Docker image:

```text
hello-world:latest
```

After running:

```bash
docker pull ubuntu
```

Docker downloaded:

```text
ubuntu:latest
```

Then I downloaded another version:

```bash
docker pull ubuntu:24.04
```

Now my local Docker host contains:

```text
hello-world:latest
ubuntu:latest
ubuntu:24.04
```

This shows that Docker can store multiple versions (tags) of the same image at the same time.

---

## Remember

- Downloads images from Docker Hub by default.
- Images are stored locally after downloading.
- If the image already exists locally, Docker will not download it again.
- If no version is specified, Docker uses the **latest** tag automatically.

---

# 4. `docker images`

## What does it do?

Lists all Docker images available on the **local Docker host**.

It displays information such as:

- Repository
- Tag (Version)
- Image ID
- Disk Usage
- Content Size

---

## Syntax

```bash
docker images
```

---

### Show only Image IDs

```bash
docker images -q
```

---

## Examples

### List all local images

```bash
docker images
```

Example output:

```text
hello-world:latest
ubuntu:latest
ubuntu:24.04
```

---

### Display only Image IDs

```bash
docker images -q
```

Example output:

```text
96498ffd522e
b7f48194d4d8
4fbb8e6a8395
```

Only the Image IDs are displayed.

---

## Remember

- Shows only images stored on your local machine.
- Does **not** download any image.
- Useful before creating or running containers.

---

# 5. `docker ps`

## What does it do?

Displays Docker containers.

By default, it shows **only the running containers**.

The output includes information such as:

- Container ID
- Image
- Command
- Status
- Ports
- Container Name

---

## Syntax

```bash
docker ps
```

---

### Show all containers

```bash
docker ps -a
```

or

```bash
docker ps --all
```

---

## Examples

### Show only running containers

```bash
docker ps
```

If no containers are running, the output will be empty.

---

### Show all containers (Running + Stopped)

```bash
docker ps -a
```

Example output:

```text
CONTAINER ID
IMAGE
STATUS
NAMES

hello-world
Exited (0)
cool_driscoll
```

This command displays both running and stopped containers.

---

## What I Observed

When I executed:

```bash
docker ps
```

nothing was displayed because there were no running containers.

However, after running:

```bash
docker ps -a
```

I could see all previously created **hello-world** containers with the following status:

```text
Exited (0)
```

This means the containers successfully completed their task and then stopped automatically.

---

## Remember

- `docker ps` → Shows only running containers.
- `docker ps -a` or `docker ps --all` → Shows both running and stopped containers.
- Very useful while checking container status and debugging.

---

# Quick Revision

| Command | Purpose |
|----------|---------|
| `docker version` | Shows Docker Client and Server versions |
| `docker -v` | Shows installed Docker version only |
| `docker info` | Displays Docker Engine information |
| `docker pull ubuntu` | Downloads the latest Ubuntu image |
| `docker pull ubuntu:24.04` | Downloads Ubuntu version 24.04 |
| `docker pull --all-tags ubuntu` | Downloads all Ubuntu image tags |
| `docker images` | Lists all Docker images stored locally |
| `docker images -q` | Displays only Image IDs |
| `docker ps` | Shows running containers |
| `docker ps -a` | Shows all containers (running + stopped) |

# Summary

In this section, I learned how to:

- Verify that Docker is installed.
- Check Docker Engine information.
- Download Docker images from Docker Hub.
- View all locally available Docker images.
- Check running and stopped Docker containers.

These commands are the foundation for working with Docker and are used frequently before creating or managing containers.
