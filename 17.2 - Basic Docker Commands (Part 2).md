# 17.2 - Basic Docker Commands (Part 2)

In this section, I learned how to create, start, stop, and interact with Docker containers. I also learned a few Linux process commands because Docker containers are simply lightweight Linux processes.

---

# 1. `docker create`

## What does it do?

Creates a Docker container in a **stopped state**.

Unlike `docker run`, this command **does not start** the container. It only prepares the container so that it can be started later.

---

## Syntax

```bash
docker create [OPTIONS] IMAGE [COMMAND] [ARG...]
```

---

## Examples

### Create a container using the latest Nginx image

```bash
docker create --name my-container nginx
```

Creates a container named **my-container** using the latest **nginx** image.

If the image is not available locally, Docker automatically downloads it from Docker Hub.

---

### Create a container using a specific Ubuntu version

```bash
docker create --name my-container ubuntu:20.04
```

Creates a stopped container named **my-container** using Ubuntu **20.04**.

---

## Remember

- Creates a container only.
- Does **not** start it.
- If the image doesn't exist locally, Docker downloads it automatically.

---

# 2. `docker start`

## What does it do?

Starts an **already created** container.

This command only works if the container already exists.

---

## Syntax

```bash
docker start [OPTIONS] CONTAINER [CONTAINER...]
```

---

## Examples

### Start by container name

```bash
docker start my-container
```

---

### Start by Container ID

```bash
docker start abc1234
```

---

### Start multiple containers

```bash
docker start container1 container2
```

---

## Remember

- Starts existing containers only.
- You can use either the container **Name** or **Container ID**.
- Multiple containers can be started together.

---

# 3. `docker stop`

## What does it do?

Stops a running Docker container **gracefully**.

Docker first asks the application to stop properly before forcefully terminating it.

By default, Docker waits **10 seconds**.

---

## Syntax

```bash
docker stop [OPTIONS] CONTAINER [CONTAINER...]
```

---

## Examples

### Stop one container

```bash
docker stop my-container
```

---

### Wait 30 seconds before stopping

```bash
docker stop -t 30 my-container
```

---

### Stop multiple containers

```bash
docker stop container1 container2
```

---

## Remember

- Stops containers gracefully.
- Default timeout = **10 seconds**.
- Use `-t` to change the waiting time.

---

# 4. Linux `ps`

Since a Docker container is simply a lightweight Linux process, knowing Linux process commands is also important.

---

## What does it do?

Shows the processes attached to your **current terminal**.

---

## Syntax

```bash
ps
```

---

## Remember

Shows only processes running in your current shell.

---

# 5. Linux `ps a`

## What does it do?

Shows terminal processes for **all users**.

---

## Syntax

```bash
ps a
```

---

## Remember

Displays terminal processes belonging to every user.

---

# 6. Linux `ps -A`

## What does it do?

Shows **every process** running on the Linux system.

---

## Syntax

```bash
ps -A
```

---

## Remember

Useful when you want to see all running processes.

---

# 7. Linux `ps x`

## What does it do?

Shows processes that are **not attached to a terminal**.

These are usually background services (daemons).

---

## Syntax

```bash
ps x
```

---

## Remember

Useful for viewing background processes.

---

# 8. Linux `ps u`

## What does it do?

Shows processes in a **user-friendly format**.

It displays:

- User
- PID
- CPU Usage
- Memory Usage
- Start Time
- Command

---

## Syntax

```bash
ps u
```

---

## Remember

Useful for identifying the owner of a process.

---

# 9. `docker exec`

## What does it do?

Executes a command **inside a running Docker container**.

The container **must already be running**.

---

## Syntax

```bash
docker exec CONTAINER COMMAND [ARG...]
```

---

## Example

```bash
docker exec my-container ls /usr/src/app
```

### Explanation

- `docker exec` → Execute a command inside a running container.
- `my-container` → Container name (or Container ID).
- `ls` → Linux command to list files.
- `/usr/src/app` → Directory to display.

---

## Remember

- Works only on **running containers**.
- Executes Linux commands inside the container.
- Container can be identified using its **Name** or **Container ID**.

---

# 10. `docker exec -t`

## What does it do?

Allocates a **TTY (Terminal)** for the command.

This allows Docker to display output through a terminal session.

---

## What is TTY?

**TTY** simply means a **terminal session**.

It lets a program display output and interact like a normal Linux terminal.

Easy trick to remember:

> **TTY = Terminal**

---

## Syntax

```bash
docker exec -t CONTAINER COMMAND
```

---

## Example

```bash
docker exec -t my-container ls /app
```

Docker executes the `ls` command and displays the output through a terminal.

---

## Remember

- `-t` = Terminal (TTY).
- Mainly used for commands that produce formatted terminal output.

---

# 11. `docker exec -i`

## What does it do?

Keeps the container's **standard input (STDIN)** open.

This allows you to send input to the running command.

---

## Syntax

```bash
docker exec -i CONTAINER COMMAND
```

---

## Example (Instructor Example)

```bash
docker exec -i my-container login /etc/hostname
```

Here, the `login` command expects user input (username and password).

The `-i` option keeps the input stream open so you can enter those values.

---

## Another Example

```bash
docker exec -i my-container cat /etc/hostname
```

This prints the hostname file from inside the container.

Although `cat` doesn't require user input, it still works with `-i`.

---

## Remember

- `-i` = Interactive Input.
- Keeps STDIN open.
- Used with commands like:

- login
- bash
- sh
- vim
- nano

or any application that requires user input.

---

# 12. `docker exec -it`

## What does it do?

Creates an **interactive terminal session** inside a running container.

It combines two options:

- `-i` → Keep input open.
- `-t` → Allocate a terminal.

Together they allow you to work inside the container like a normal Linux machine.

---

## Syntax

```bash
docker exec -it CONTAINER /bin/bash
```

---

## Example

```bash
docker exec -it my-container /bin/bash
```

This opens a Bash shell inside the running container.

Now you can execute Linux commands directly.

---

### Another Example

```bash
docker exec -it my-container cat /etc/shells
```

Displays all available shells inside the container.

---

## Remember

- `-it` = Interactive Terminal.
- Most commonly used to enter a running container.

---

# 13. `docker run`

## What does it do?

Creates **and starts** a Docker container in a single command.

Instead of running multiple commands separately, Docker performs everything automatically.

---

## Easy Way to Remember

```text
docker run
      =
docker pull
      +
docker create
      +
docker start
```

Docker automatically performs these steps:

1. Looks for the image locally.
2. If not found, downloads it from Docker Hub.
3. Creates the container.
4. Starts the container.

---

## Syntax

```bash
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
```

---

## Examples

### Run the latest Nginx image

```bash
docker run nginx
```

Creates and starts a container using the latest nginx image.

---

### Run Ubuntu 20.04 with a custom name

```bash
docker run --name cont1 ubuntu:20.04
```

Creates and starts a container named **cont1**.

---

### Run in detached mode

```bash
docker run -d nginx
```

Runs the container in the background.

---

## What is Detached Mode?

Detached mode (`-d`) means Docker runs the container in the background and immediately returns your terminal.

The container continues running even after your command finishes.

---

## Remember

- `docker run` = Pull + Create + Start.
- Downloads the image automatically if required.
- `-d` runs the container in the background.

---

# 14. `docker run -it`

## What does it do?

Creates a new container and immediately opens an interactive terminal inside it.

This is one of the most frequently used Docker commands while learning Docker.

---

## Syntax

```bash
docker run -it ubuntu /bin/bash
```

---

## Example

```bash
docker run -it ubuntu /bin/bash
```

### Explanation

- `docker run` → Create and start a container.
- `-i` → Keep input open.
- `-t` → Allocate a terminal.
- `ubuntu` → Use Ubuntu image.
- `/bin/bash` → Open the Bash shell.

After executing this command, you enter the Ubuntu container directly.

---

### Another Example

```bash
docker run -it ubuntu bash
```

Works the same if **bash** is already available in the container's **$PATH**.

---

## Remember

- Creates a new container.
- Starts the container.
- Opens an interactive Bash terminal.
- Most commonly used while practicing Docker.

# Quick Revision

| Command | Purpose |
|----------|---------|
| `docker create` | Create a stopped container |
| `docker start` | Start an existing container |
| `docker stop` | Gracefully stop a container |
| `ps` | Show processes of the current terminal |
| `ps a` | Show terminal processes of all users |
| `ps -A` | Show all system processes |
| `ps x` | Show background processes |
| `ps u` | Show processes with owner details |
| `docker exec` | Execute a command inside a running container |
| `docker exec -t` | Execute a command with a terminal (TTY) |
| `docker exec -i` | Keep standard input open |
| `docker exec -it` | Open an interactive terminal inside a running container |
| `docker run` | Pull + Create + Start a container |
| `docker run -it` | Create, start, and enter a container |

In this section, I learned how to:

- Create Docker containers.
- Start and stop containers.
- Execute commands inside running containers.
- Open an interactive shell inside a container.
