# 17.3 - Basic Docker Commands (Part 3)

In this section, I learned two useful Docker commands:

- `docker run --rm`
- `docker inspect`

These commands are commonly used for running temporary containers and inspecting Docker objects.

---

# 1. `docker run --rm`

## What does it do?

The `--rm` option tells Docker to **automatically remove the container** after it finishes running.

It is useful for **temporary (one-time) containers** that are only needed for a short task, such as testing or learning Docker.

---

## Syntax

```bash
docker run --rm IMAGE COMMAND [ARG...]
```

---

## Example 1

```bash
docker run --rm ubuntu echo "this will self-destruct"
```

### Explanation

- `docker run` → Create and start a container.
- `--rm` → Automatically delete the container after it exits.
- `ubuntu` → Use the Ubuntu image.
- `echo` → Print a message.
- `"this will self-destruct"` → Text displayed on the terminal.

### What happens?

1. Docker creates a temporary Ubuntu container.
2. Executes the `echo` command.
3. Displays the message.
4. Stops the container.
5. Automatically removes the container.

Once the command finishes, the container no longer exists.

---

## Example 2

```bash
docker run -it --rm ubuntu bash
```

### Explanation

- `-i` → Keep standard input (STDIN) open.
- `-t` → Allocate a pseudo terminal (TTY).
- `--rm` → Remove the container after it exits.
- `ubuntu` → Use the Ubuntu image.
- `bash` → Start an interactive Bash shell.

### What happens?

1. Docker creates and starts a new Ubuntu container.
2. Opens an interactive Bash terminal.
3. You can execute Linux commands inside the container.
4. When you type:

```bash
exit
```

Docker stops the container and automatically deletes it.

---

## When should I use `--rm`?

Use it for short-lived containers, such as:

- Learning Docker
- Testing commands
- Running `curl`
- Running `ping`
- Executing small scripts
- Quick experiments
- Temporary troubleshooting

---

## When should I NOT use `--rm`?

Avoid using it for containers that should continue running, such as:

- Nginx or Apache web servers
- MySQL or PostgreSQL databases
- Backend APIs
- Production applications
- Long-running services

---

## Remember

- `--rm` automatically deletes the container after it exits.
- Best for temporary or one-time tasks.
- Can be combined with options like `-it`.

---

## Quick Revision

| Command | Purpose |
|---------|---------|
| `docker run --rm ubuntu echo "Hello"` | Run a temporary container and delete it automatically |
| `docker run -it --rm ubuntu bash` | Open a temporary interactive Ubuntu shell and delete it after exiting |

---

## Summary

The `--rm` option automatically removes a container after it finishes running. It is ideal for testing, learning Docker, and executing one-time commands without leaving unused containers on the system.

# 2. `docker inspect`

## What does it do?

The `docker inspect` command displays **detailed, low-level information** about a Docker object.

The output is returned in **JSON format**, containing almost everything Docker knows about that object.

You can inspect:

- Containers
- Images
- Volumes
- Networks

Unlike `docker exec`, the container **does not need to be running**. You can inspect both **running** and **stopped** containers.

---

## Syntax

```bash
docker inspect [OPTIONS] OBJECT [OBJECT...]
```

Where **OBJECT** can be:

- Container Name
- Container ID
- Image Name
- Volume Name
- Network Name

---

## Example 1

```bash
docker inspect my-container
```

### Explanation

Displays detailed information about the container, including:

- Container Name
- Container ID
- Current Status
- IP Address
- Network Configuration
- Mounted Volumes
- Environment Variables
- Labels
- Creation Time
- Image Used
- And much more

---

## Example 2

```bash
docker inspect nginx:latest
```

### Explanation

Displays detailed information about the **nginx:latest** image, including:

- Image ID
- Repository
- Tags
- Creation Date
- Layers
- Architecture
- Operating System
- Size
- Configuration

---

## Why is it useful?

`docker inspect` is commonly used to:

- Find a container's IP address.
- Check mounted volumes.
- View environment variables.
- View network configuration.
- Debug Docker containers.
- Inspect image metadata.

---

## Remember

- Returns detailed information in **JSON format**.
- Works with **containers, images, volumes, and networks**.
- Supports both **running and stopped** containers.
- It is a read-only command—it only displays information.

---

## Quick Revision

| Command | Purpose |
|---------|---------|
| `docker inspect my-container` | Show complete information about a container |
| `docker inspect nginx:latest` | Show complete information about an image |

---
