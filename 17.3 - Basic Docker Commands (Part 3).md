# 17.3 - Basic Docker Commands (Part 3)

In this section, I learned two useful Docker commands:

- `docker run --rm`
- `docker inspect`

These commands are commonly used for running temporary containers and inspecting Docker objects.

---

# 1. `docker run --rm`

## What does it do?

The `--rm` option tells Docker to **automatically remove the container** after it finishes running.

It is useful for **temporary (one-time) containers** that are only needed for a short task, such as testing or learning Docker.

---

## Syntax

```bash
docker run --rm IMAGE COMMAND [ARG...]
```

---

## Example 1

```bash
docker run --rm ubuntu echo "this will self-destruct"
```

### Explanation

- `docker run` → Create and start a container.
- `--rm` → Automatically delete the container after it exits.
- `ubuntu` → Use the Ubuntu image.
- `echo` → Print a message.
- `"this will self-destruct"` → Text displayed on the terminal.

### What happens?

1. Docker creates a temporary Ubuntu container.
2. Executes the `echo` command.
3. Displays the message.
4. Stops the container.
5. Automatically removes the container.

Once the command finishes, the container no longer exists.

---

## Example 2

```bash
docker run -it --rm ubuntu bash
```

### Explanation

- `-i` → Keep standard input (STDIN) open.
- `-t` → Allocate a pseudo terminal (TTY).
- `--rm` → Remove the container after it exits.
- `ubuntu` → Use the Ubuntu image.
- `bash` → Start an interactive Bash shell.

### What happens?

1. Docker creates and starts a new Ubuntu container.
2. Opens an interactive Bash terminal.
3. You can execute Linux commands inside the container.
4. When you type:

```bash
exit
```

Docker stops the container and automatically deletes it.

---

## When should I use `--rm`?

Use it for short-lived containers, such as:

- Learning Docker
- Testing commands
- Running `curl`
- Running `ping`
- Executing small scripts
- Quick experiments
- Temporary troubleshooting

---

## When should I NOT use `--rm`?

Avoid using it for containers that should continue running, such as:

- Nginx or Apache web servers
- MySQL or PostgreSQL databases
- Backend APIs
- Production applications
- Long-running services

---

## Remember

- `--rm` automatically deletes the container after it exits.
- Best for temporary or one-time tasks.
- Can be combined with options like `-it`.

---

## Quick Revision

| Command | Purpose |
|---------|---------|
| `docker run --rm ubuntu echo "Hello"` | Run a temporary container and delete it automatically |
| `docker run -it --rm ubuntu bash` | Open a temporary interactive Ubuntu shell and delete it after exiting |

---

## Summary

The `--rm` option automatically removes a container after it finishes running. It is ideal for testing, learning Docker, and executing one-time commands without leaving unused containers on the system.

# 2. `docker inspect`

## What does it do?

The `docker inspect` command displays **detailed, low-level information** about a Docker object.

The output is returned in **JSON format**, containing almost everything Docker knows about that object.

You can inspect:

- Containers
- Images
- Volumes
- Networks

Unlike `docker exec`, the container **does not need to be running**. You can inspect both **running** and **stopped** containers.

---

## Syntax

```bash
docker inspect [OPTIONS] OBJECT [OBJECT...]
```

Where **OBJECT** can be:

- Container Name
- Container ID
- Image Name
- Volume Name
- Network Name

---

## Example 1

```bash
docker inspect my-container
```

### Explanation

Displays detailed information about the container, including:

- Container Name
- Container ID
- Current Status
- IP Address
- Network Configuration
- Mounted Volumes
- Environment Variables
- Labels
- Creation Time
- Image Used
- And much more

---

## Example 2

```bash
docker inspect nginx:latest
```

### Explanation

Displays detailed information about the **nginx:latest** image, including:

- Image ID
- Repository
- Tags
- Creation Date
- Layers
- Architecture
- Operating System
- Size
- Configuration

---

## Why is it useful?

`docker inspect` is commonly used to:

- Find a container's IP address.
- Check mounted volumes.
- View environment variables.
- View network configuration.
- Debug Docker containers.
- Inspect image metadata.

---

## Remember

- Returns detailed information in **JSON format**.
- Works with **containers, images, volumes, and networks**.
- Supports both **running and stopped** containers.
- It is a read-only command—it only displays information.

---

## Quick Revision

| Command | Purpose |
|---------|---------|
| `docker inspect my-container` | Show complete information about a container |
| `docker inspect nginx:latest` | Show complete information about an image |

# 3. `docker rmi`

## What does it do?

The `docker rmi` command removes one or more Docker images from your local system.

It only deletes the image stored on your local Docker host. It does **not** remove the image from Docker Hub.

---

## Syntax

```bash
docker rmi [OPTIONS] IMAGE [IMAGE...]
```

---

## Example 1

```bash
docker rmi my-image
```

### Explanation

Removes the Docker image named **my-image** from the local system.

---

## Example 2

```bash
docker rmi image1 image2 image3
```

### Explanation

Removes multiple Docker images at once.

---

## Example 3

```bash
docker rmi -f my-image
```

### Explanation

- `-f` → Force remove the image.

This is useful when the image is still being referenced by stopped containers.

---

## Remember

- Removes images only from your local machine.
- Does not delete anything from Docker Hub.
- Use `-f` only when necessary.

---

## Quick Revision

| Command | Purpose |
|---------|---------|
| `docker rmi my-image` | Remove one image |
| `docker rmi image1 image2` | Remove multiple images |
| `docker rmi -f my-image` | Force remove an image |

# 4. `docker logs`

## What does it do?

The `docker logs` command displays the output (logs) generated by a container.

It is commonly used for troubleshooting, debugging, and monitoring container activity.

---

## Syntax

```bash
docker logs [OPTIONS] CONTAINER
```

---

## Example 1

```bash
docker logs my-container
```

### Explanation

Displays all logs generated by **my-container**.

---

## Example 2

```bash
docker logs -f my-container
```

### Explanation

- `-f` → Follow the logs in real time.

The terminal keeps updating as new log messages are generated.

This is similar to Linux's `tail -f`.

---

## When is it useful?

Use it when:

- Debugging applications
- Checking errors
- Monitoring a running container
- Viewing application output

---

## Remember

- Shows container logs.
- `-f` follows logs in real time.
- Helpful for debugging and monitoring.

---

## Quick Revision

| Command | Purpose |
|---------|---------|
| `docker logs my-container` | Show container logs |
| `docker logs -f my-container` | Follow logs continuously |

---

# 5. `docker --help`

## What does it do?

The `docker --help` command displays Docker's built-in help documentation.

It lists all available Docker commands along with a short description of each.

---

## Syntax

```bash
docker --help
```

---

## Example 1

```bash
docker --help
```

### Explanation

Shows all available Docker commands, categories, and descriptions.

---

## Example 2

```bash
docker run --help
```

### Explanation

Displays detailed help for the `docker run` command, including all available options and flags.

---

## When is it useful?

Use it whenever you forget:

- Command syntax
- Available options
- Flags
- Supported arguments

---

## Remember

- `docker --help` → Help for Docker.
- `docker <command> --help` → Help for a specific command.

---

## Quick Revision

| Command | Purpose |
|---------|---------|
| `docker --help` | Show all Docker commands |
| `docker run --help` | Show help for `docker run` |

---

# 6. `docker image prune`

## What does it do?

The `docker image prune` command removes unused Docker images from the local system to free up disk space.

It only removes images that are no longer being used by any container.

---

## Syntax

```bash
docker image prune [OPTIONS]
```

---

## Example

```bash
docker image prune
```

### Explanation

Removes all unused Docker images from the local Docker host.

Docker will ask for confirmation before deleting the images.

---

## Why is it useful?

Over time, unused images consume disk space.

Running this command helps keep your Docker environment clean and saves storage.

---

## Remember

- Removes only unused images.
- Frees disk space.
- Safe for images currently used by containers.

---

## Quick Revision

| Command | Purpose |
|---------|---------|
| `docker image prune` | Remove unused Docker images |
